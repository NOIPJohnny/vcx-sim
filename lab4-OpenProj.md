# 图形学物理仿真 Tutorial for Lab 4

## 概览
Lab 4 为本课程的综合性大作业（Final Project），旨在让同学们将所学的物理仿真方法进行整合，并探索更复杂或更开放的问题。
- 分组形式：3–4 人一组
- 提交内容：代码 + 实验报告 + Git记录
- 课堂展示（Demo + 简短讲解）

可以：
- 从推荐项目中选择
- 或自行提出选题（需提前与助教/老师确认）

## 评分原则
1. 技术难度（25%）  
   是否挑战复杂问题（如多物理耦合 / 逆问题 / learning）  
   是否有合理的技术选型
2. 实现质量（25%）  
   仿真是否稳定（不爆炸、不明显穿透）  
   代码结构是否清晰  
   是否有合理优化  
3. 交互与展示（25%）  
   是否提供 UI（鼠标交互 / 参数调节等）  
   Demo 是否直观、有说服力  
4. 报告质量（25%）  
   是否清晰说明：  
   - 方法原理  
   - 实现思路  
   - 遇到的问题与解决方案  
   - 是否明确说明每个人的分工  
5. 注意：
   - 推荐使用课程框架，或利用Taichi，如使用其他框架需提供编译说明，或与助教联系进行线下的评定。
   - 直接提交开源代码或者做很小的修改将获得非常低的分数。
   - 如果代码复现和开源项目（比如基于webgl的Ten Minute Physics等）相似度过高，需要在报告提供详细的工作内容描述以避免减分。
   - 请继续确保使用 Git 管理你的代码，在完成 Lab4 时输出提交记录，也可进一步规避上述情形。


## 建议的project

1. 异构的多物理仿真系统：  
   你需要将不同物理系统进行耦合：刚体，软体（FEM / XPBD），流体（Grid / PIC / FLIP）等。可选取2种或三种都选，建议预留更多时间整合多物理耦合的demo。以下是一些建议：

   - 实现支持碰撞的软体：建议参考xpbd，当然也可选ipc方案，但前者实时性更好。  
      xpbd建议参考 https://matthias-research.github.io/pages/tenMinutePhysics/index.html 的一系列课程。

   - 复杂刚体系统： 
      1. 【简单版本】Ten-Minute Physics Lesson22  
      在lab1的基础上，为刚体之间添加一些约束。
      可以参考：https://matthias-research.github.io/pages/tenMinutePhysics/index.html  
      基于xpbd的实现，建议叠加lab1的碰撞，获得一个支持碰撞的xpbd复杂刚体系统  
      注意：lab1的碰撞叠加xpbd并不能获得一个完全无穿透的仿真结果，但至少有一定的碰撞解除效果  
      2. 【比较困难】基于impulse的铰接刚体系统实现  
      可以参考：论文 https://animation.rwth-aachen.de/media/papers/2006-VRIPHYS-FastDynamicSimulation.pdf  
      项目也有开源代码（c++但历史久远） https://www.animation.rwth-aachen.de/software/ibds-physics-library/   
      也可结合 https://interactivecomputergraphics.github.io/physics-simulation/examples/ibds.html  
      注意：这一实现涉及到矩阵求逆，论文中推荐LU分解，如果是复杂的大型铰接系统，笔记本可能会有计算效率不满足实时的问题，建议从简单的开始（比如双摆、三摆）
      3. 【有趣的扩展】可以构建铰接刚体的人体骨骼，读入一个主观动作序列，同时给人体穿上服装，或者给周边放置自由刚体，观察人体主观动作和被动物理（服装、自由刚体）之间的交互。建议服装考虑pbd、xpbd等实时的方法。

   - 固体流体耦合：  
      1.【简单版本】耦合Lab1的刚体和 Lab2的流体  (08.pptx, p53)
      - 首先将流体上一步结算的压强作为刚体/软体的外力项  
         可以先遍历流体grid cell, 确定它是否在哪个刚体内部。  
         再遍刚体内部的cell，确定它们是否是流固接触面的cell  
         最后累计所有流固接触面的cell上的压力
      - 然后将刚体的边界条件作为流体当前解算的边界条件   
         将流体粒子移出刚体内部
      - UI扩展：支持鼠标给某个刚体施加外力，进而产生更多的流固耦合交互

      2.【比较困难】流体耦合软体（FLIP × FEM），可以实现软体小熊落在水中等效果
      更建议使用 FLIP 的hybrid结构，通过类似MPM的方式，在 **grid 上进行动量交换**，避免显式碰撞检测，建议加入 damping 提升稳定性

      3.【困难】可能的扩展：实现 A fast variational framework for accurate solid-fluid coupling https://dl.acm.org/doi/10.1145/1276377.1276502  

      4.【困难】考虑流固耦合中，流体的表面张力现象。这要求对流体的表面进行建模。


2. MPM 的同构多物理系统：
   1. 如使用我们的代码框架，则搭建一套包含流体、软体、近似刚体效果的MPM仿真器已是比较大的工作量。
   2. 如果使用taichi框架，因为会大大降低MPM的实现复杂度，建议额外增加 MPM和拉格朗日系统（如xpbd的布料）的耦合，或考虑增加破碎等现象。
   3. 同样建议预留更多时间整合多物理耦合的demo

3. 物理仿真驱动的游戏项目：  
   基于课程中的物理仿真方法，实现一个具有交互性的小游戏
   - 要求：将物理仿真（刚体 / 软体 / 流体）作为游戏机制的一部分，稳定性是这个工作的重要评分点，巧妙的关卡设计也是能够加分的因素。
   - 示例：
      - 三维弹球（Pinball）
      - 三维box中的愤怒的小鸟（弹射 + 刚体破坏）
      - 2D 小游戏（如水流引导 / “小鳄鱼爱洗澡”类）
      - 具有物理反馈的俄罗斯方块（堆叠稳定性 / 碰撞）
      - 复杂系统如小车的物理驱动行驶

4. 探索物理仿真和3DGS的融合：  
   可以参考：https://github.com/XPandora/PhysGaussian   
   建议使用taichi框架, 建议有3DGS基础的同学选择。
   

5. 探索仿真+learning的问题：  
   工作包括设计Learning学习的目标，仿真完成数据生成、根据目标完成网络设计和训练等，可对比网络和仿真的效率和效果。  
   示例任务：比如训练一个从速度场预测压强场，用于解除速度场散度的神经网络。  
   建议使用taichi，warp等。
   
6. 探索逆向仿真问题:  
   尝试流体、xpbd软体、或MPM软体的 控制 / 反问题，如优化初始条件或外力，使流体/软体达到目标状态。
   可参考：https://zhuanlan.zhihu.com/p/527112831  
   建议使用taichi框架




## 作业提交
作业须提交到**教学网**。

1. 代码管理：请确保你使用 Git 管理你的代码，在完成 Lab1 时输出你的提交记录：  
   ```bash
   git log --stat > lab4_log.txt
   ```
   log中应当看到多人合作。

2. 清理记录：提交前执行 (VCX) 
   ```bash 
   xmake clean -a
   ```  
   或删除 __pycache__ (Taichi)。
   来移除所有二进制和临时文件。

3. 同时，也请提交一份作业报告，简要说明核心代码的思路，实现的交互方法，并展示 Demo。
   报告中应有一段文字说明合作分工情况。

4. 将输出的提交记录、源代码和报告打包成 zip 压缩包，并以`lab4_<student1-ID>_<student2ID>_<student3-ID>_<student4-ID>.zip`命名，提交到教学网。
